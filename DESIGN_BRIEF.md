# GeoMonitor 리디자인 브리프

## 1. 한 줄 정의
지반 계측 센서(수위계·지중경사계 등)의 실시간 데이터를 24시간 모니터링하고, 임계값 기반 알람에 대응하는 산업용 웹 시스템.

---

## 2. 리디자인의 이유 (가장 중요)
현재 디자인은 기능은 모두 갖추어져 있지만, **첫 진입 시 "이게 뭔지" 인지하기 어렵다**. 화면에 정보가 동시에 너무 많아서 시선 진입점이 없고, 무엇이 중요한지가 시각적으로 구분되지 않는다.

**해결해야 할 핵심**: 화면을 처음 본 사용자가 **3초 안에 "지금 안전한가? 내가 봐야 할 게 뭔가?"** 를 답할 수 있어야 한다.

---

## 3. 사용 맥락 — 앰비언트 모니터링(Ambient Monitoring)
사용자가 **항상 켜두고 수시로 스쳐 보는 도구**. CCTV/NOC 모니터 카테고리.

이로 인한 제약:
- **눈 부담 최소화** → 다크 모드 기본값 권장
- **평시엔 조용히, 이상 시엔 강하게** → 정상 상태는 채도 낮은 그레이/블루, 이상 상태만 채도 높은 컬러
- **멀리서도 식별** → 핵심 숫자·배지 크기 크게, 본문은 그 다음
- **갱신 시 깜빡임 금지** → 60초 폴링 시 부드러운 전환

---

## 4. 디자인 톤 — 산업용 + 트렌디
한 줄 요약:
> **"NOC에서 24시간 켜둬도 피곤하지 않으면서, IR 자료에 스크린샷 써도 되는 톤"**

레퍼런스: **Linear, Vercel/Grafana 최신, Datadog/Honeycomb, Tesla 차량 디스플레이**

---

## 5. 핵심 디자인 원칙
1. **3초 룰** — 화면 진입 3초 안에 전체 상태 + 가장 시급한 1건이 보여야 함
2. **시각적 위계 1순위 = 상태** — 색·크기·위치 모두 위험 > 주의 > 정상 순으로 시선 이동
3. **컬러는 의미만, 장식 금지** — 빨강/노랑/초록은 상태 전용. 브랜드 컬러가 위험 색을 침범하지 않음
4. **각 화면 = 한 가지 목적** — 대시보드는 "감시", 센서 상세는 "분석", 알람은 "대응"

---

## 6. 타깃 디바이스
- **데스크탑 1순위 (1440~1920)** — 모니터링 도구의 본진
- **모바일은 "알람 확인 + 단일 센서 조회"만** — 풀 기능 모바일화 X. 알람 받고 → 모바일 확인 → 데스크탑에서 처리하는 흐름이면 충분

---

## 7. MVP 진행 순서 (페이지별 따로 디자인 시 컴포넌트 불일치 방지)
1. **디자인 시스템 파운데이션** (컬러·타이포·컴포넌트) + **대시보드** 1장
2. 톤 확정 후 → **센서 상세**
3. → **알람 관리**
4. 이후 나머지 9개 화면

---

## 9. 화면별 상세

### 9-1. 대시보드 (MVP 1순위)
**목적**: 전체 상태를 한눈에 파악 + 위험 즉시 인지

**표시 데이터**
- 카카오맵 + 현장별 마커/인포윈도우
- 현장 목록 (현장명·위치)
- KPI 4종: 전체/정상/주의/위험/오프라인 센서 수
- 위험 센서 배너 (danger 상태 리스트)
- 최근 알람 5건 (severity·센서·메시지·시각)
- 상태 필터 패널 (KPI 클릭 시 해당 상태 센서 목록)

**사용자 액션**: 마커 클릭→현장 이동 / KPI 카드 클릭→상태 필터 / 센서 행 클릭→상세 이동 / 새로고침

**실시간**: 60초 자동 갱신
**권한 분기**: 없음 (모두 동일)

---

### 9-2. 센서 상세 (MVP 2순위)
**목적**: 단일 센서 데이터 깊이 분석 + 보정/관리

**구조**: 좌측 센서 트리 + 중앙 콘텐츠 + 우측 정보 패널

**중앙 콘텐츠**
- 헤더: 센서명, 현재값, 상태, 마지막 측정시각
- **평면도**: 배경 이미지 + 센서 아이콘 (드래그로 위치 조정)
- **트렌드 차트 (Recharts)**: 시간별/일별 모드, 임계값 라인
- **실시간 요약 패널** (드래그·최소화 가능): 누적 변화량/현재값/배터리/알람 수
- **측정 로그 테이블**: 측정일/경과일/값/변화량

**우측 패널**: 설치 정보, 임계값, 1차/2차 관리기준, 보정 파라미터

**사용자 액션**: 날짜·시간·모드·depth 선택 / 차트 줌 / 평면도 아이콘 드래그·추가·삭제·라벨 수정 / 보정값·관리기준 편집 / QR 생성·다운로드 / **Excel·PDF 내보내기 (차트 이미지 포함, Water Level Report 양식)** / 인쇄

**실시간**: 날짜/조건 변경 시 즉시 재조회
**권한 분기**: MultiMonitor → 평면도 드래그·보정값 수정·모든 편집 버튼 X (완전 읽기 전용)

**특수 처리**: WL-02 (센서코드 80053, depth=2)는 depth 1·3 평균 자동 산출

---

### 9-3. 알람 관리 (MVP 3순위)
**목적**: 발생 알람에 빠르게 대응

**표시 데이터**: 알람 카드 (severity 배지, 센서 코드/명, 현장, 메시지, 측정값/기준값, 발생시각, 확인자, 처리 상태)

**사용자 액션**: severity 필터(전체/위험/주의) / 개별 확인 / 전체 확인(일괄) / 센서 보기 이동

**실시간**: 페이지 진입 시 1회 + 사이드바 미확인 알람 배지는 60초마다 갱신
**권한 분기**: MultiMonitor → 확인 버튼 X

---

### 9-4. 센서 관리 (2차) — 4개 내부 탭
- **(1) 모니터**: 센서 목록 (코드·명·현장·관리번호·설치일·현재값·상태·마지막 측정·배터리), 상태/현장 필터
- **(2) 관리**: 센서 추가/편집/삭제. **등록 모달은 11개 섹션** (식별·분류·기본정보·단위·임계값·동작·수식/파라미터·관리기준·설치정보·상세·비고) — 모달 디자인이 매우 복잡, **별도 설계 필요**
- **(3) 계산식**: 이름·표현식·설명 CRUD
- **(4) 재수집**: 재수집 요청 목록, 에이전트 온라인/오프라인 상태

**권한 분기**: MultiMonitor → 모든 액션 X / 계산식 관리는 Operator 이상만

---

### 9-5. 현장 관리 (2차)
**표시 데이터**: 현장 카드 (현장명·위치, 상태 비율 바, 정상/주의/위험/오프라인 수, 담당자 목록)
**사용자 액션**: 상태 필터 / 현장 추가·편집(평면도 업로드·담당자·센서 배정·지오코딩)·삭제
**권한 분기**: MultiMonitor·Monitor → 추가/편집/삭제 X

---

### 9-6. 사용자 관리 (3차)
**표시 데이터**: 사용자 테이블 (ID·이메일·핸드폰·권한·가입일·마지막 로그인·상태)
**사용자 액션**: 상태 필터 / 사용자 추가·수정·비활성·활성·삭제
**권한 분기**: **Administrator 전용 페이지**

---

### 9-7. 보조 화면
- **로그인**: 로그인/회원가입 탭, 테스트 계정 자동 입력
- **QR 뷰**: 센서 QR 단독 화면 (사이드바 X, 외부 스캔용)
- **파일 관리**: 기본 골격만 — 표준 패턴 적용
- **사이드바 (공통)**: 네비게이션 + 미확인 알람 배지(60초 갱신) + 비밀번호 변경 모달

---

## 10. 실시간 갱신 정책
HTTP Polling 방식 (WebSocket/SSE 미사용):

| 영역 | 주기 |
|------|------|
| 대시보드 전체 | 60초 자동 |
| 사이드바 알람 배지 | 60초 자동 |
| 센서 상세 측정 데이터 | 사용자 액션 시 (날짜·조건 변경) |
| 기타 목록 | 페이지 진입 시 1회 |

**디자인 시 고려**: 60초마다 데이터 바뀔 때 깜빡임 없이 부드럽게 전환.

---

## 11. 권한 체계
5단계 권한: **Administrator · Manager · Operator · Monitor · MultiMonitor(뷰어)**

핵심 분기 규칙: **MultiMonitor는 모든 편집/추가/삭제/알람 확인 불가** — 뷰어용 별도 화면이 아닌, **같은 화면에서 액션 버튼만 사라지는 구조**.

| 기능 | Admin | Manager | Operator | Monitor | MultiMonitor |
|------|:-:|:-:|:-:|:-:|:-:|
| 조회 전체 | O | O | O | O | O |
| 센서 추가/편집/삭제 | O | O | O | O | **X** |
| 평면도 아이콘 편집 | O | O | O | O | **X** |
| 보정값·관리기준 수정 | O | O | O | O | **X** |
| 계산식 관리 | O | O | O | **X** | **X** |
| 재수집 요청 | O | O | O | O | **X** |
| 알람 확인 | O | O | O | O | **X** |
| 현장 추가/편집/삭제 | O | O | O | **X** | **X** |
| 사용자 관리 | O | **X** | **X** | **X** | **X** |
| 사이드바 "기타" 섹션 | O | O | O | O | **숨김** |

## 12. 기술 제약 (구현 시 참고)
- **React 19 + Next.js 16 (App Router) + Tailwind CSS 4**
- 차트: **Recharts** (선/영역 그래프)
- 지도: **카카오맵 API**
- 한글 가독성 우선

---

## 13. 디자인 산출물 요청 순서

1. **디자인 시스템 파운데이션**
   - 컬러 팔레트 (브랜드/상태/뉴트럴)
   - 타이포그래피 스케일
   - 기본 컴포넌트 (버튼·배지·카드·테이블·모달·필터·차트 컨테이너)
   - 다크/라이트 모드 정책
2. **대시보드** (데스크탑 + 모바일)
3. **센서 상세** (데스크탑 + 모바일은 핵심 정보만)
4. **알람 관리** (데스크탑 + 모바일)

---

> 위 1~4번 받은 후 톤·시스템 확정 → 나머지 9개 화면 일괄 진행 권장.
