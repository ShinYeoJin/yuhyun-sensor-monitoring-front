# GeoMonitor 프론트엔드

> 지반 계측 센서 실시간 모니터링 시스템 - 프론트엔드

## 📋 프로젝트 개요

GeoMonitor는 지반 계측 센서 데이터를 실시간으로 수집·저장·시각화하는 웹 기반 모니터링 시스템입니다.
수위계, 지중경사계 등 다양한 센서의 측정값을 1시간 단위로 자동 수집하며, 임계값 초과 시 알람을 발생시킵니다.

## 🔗 배포 URL

- **프론트엔드**: https://yuhyun-sensor-monitoring-front.vercel.app
- **백엔드 API**: https://yuhyun-sensor-monitoring-back.onrender.com
- **Swagger UI**: https://yuhyun-sensor-monitoring-back.onrender.com/api-docs

## 🛠 기술 스택

- **Framework**: Next.js 16
- **Language**: TypeScript
- **Styling**: Tailwind CSS
- **폰트**: 맑은 고딕 (Malgun Gothic) — v1.5.0 적용
- **차트**: Recharts
- **수식 계산**: mathjs (계산식 미리보기용 — v1.5.0)
- **PDF 출력**: jsPDF + jspdf-autotable
- **엑셀 출력**: ExcelJS
- **배포**: Vercel

## 📱 주요 기능

### 대시보드
- 전체 센서 현황 KPI 카드 (전체/정상/주의/위험/오프라인)
- 최근 알람 목록
- 1분마다 자동 갱신, 새로고침 버튼 (갱신 중 상태 표시)
- **데이터 수신 지연 경고**: 2시간 이상 미수신 시 경고 배너 자동 표시/해제 (센서명 기준)
- **카카오맵 플레이스홀더**: 카카오 계정 연동 후 현장 위치 및 센서 아이콘 표시 예정 — v1.5.0

### 센서 관리
- 모니터링 / 센서 정의 / 계산식 관리 / 재수집 4개 탭
- **센서 목록 표시**: 관리번호 제거, 센서명만 표시 (모니터링 탭 / 센서 정의 탭 동일)
- 상태 필터 (전체/정상/주의/위험/오프라인)
- **데이터 수신 지연 경고**: 2시간 이상 미수신 센서 경고 표시 (센서명 기준, 대시보드와 동일)
- 센서 상세 페이지, QR 코드
- **계산식 관리 탭**: 관리자만 추가/수정/삭제, variables(변수 설명) 컬럼 표시 — v1.5.0
- **재수집 탭**: 에이전트 온라인/오프라인 상태 확인, 재수집 요청 등록 및 이력 조회
- **센서 편집 모달**: 계수값 저장 후 재편집 시 정상 불러오기
- **80053 편집 모달 관리기준**: depth별(1/2/3번) 개별 상하한 입력 UI, depth 이름은 평면도 아이콘 label 우선 표시
- **센서 정의 탭 설치일 형식**: `2026. 04. 07.` 형태 표시 — v1.5.0

### 센서 편집 모달 — 수식 및 파라미터 설정 (v1.5.0 전면 개편)
- **계산식 선택**: formulas 목록 드롭다운 + `+ 직접 입력` 옵션
  - 직접 입력 시 `customExpression` 별도 state 관리 (입력 중 창 사라짐 방지)
  - 저장 시 계산식 목록 자동 추가
- **depth별 파라미터 설정 토글**: ON 시 depth별 독립 파라미터 입력, 동적 추가/삭제
- **초기값(I) 설정**: 자동(최초 수신값) / 수동 입력 선택
- **저장 전 테스트 계산 미리보기**: 원시값 R 입력 → 실시간 계산 결과 표시
- formula_params 새 구조 `{ "1": {G, K, A, B, C}, ... }` 로 저장 (formula_id 함께 저장)
- 선택 항목의 기존 계산식 드롭다운 제거 (수식 및 파라미터 설정으로 통합)

### 센서 상세 페이지 — 레이아웃 (v1.5.0 개편)
- **전체 구조**: 상단 2단(센서정보+평면도) + 하단 시간별 트렌드 + 최하단 측정 데이터 로그
  (기존 3단 레이아웃에서 변경)
- **좌측 패널**: 센서 정보
  - 관리번호: 평면도 아이콘 클릭 시 해당 아이콘 이름 표시, 직접 진입 시 빈칸
  - 센서명/현장/설치위치/설치일/측정단위/측정주기/계산식
  - **마지막 수신**: NonMultiMonitor 역할만 표시 (MultiMonitor 숨김)
  - **1차 관리기준 인라인 편집**: 수정 버튼 클릭 → 상한/하한값 직접 입력 → 저장 즉시 반영
    - 80053: depth별 개별 상하한 설정, 일반 센서: 단일 상하한 설정
    - MultiMonitor 역할은 읽기 전용
  - 계산식 상수값 (formulaParams 있는 모든 센서 — v1.5.0)
- **중앙 패널**: 계측계획 평면도 + 센서 아이콘
  - **드래그 이동 안내**: NonMultiMonitor 역할만 표시 (MultiMonitor 숨김)
  - **아이콘 추가/수정/삭제**: 버튼으로 관리, 드래그앤드롭 위치 이동, DB 저장
  - 아이콘 클릭 시 해당 센서 전환 (80053은 depth 전환) + 관리번호 자동 표시
  - 아이콘 이름: 시간별 트렌드 depth 버튼 / 측정 데이터 로그 / QR 페이지 연동 표시
  - **평면도 변경**: PNG/JPG/PDF 업로드 (PDF 자동 PNG 변환)
  - 평면도는 현장 단위 통일 저장
  - 상태별 아이콘 색상 (정상: 초록 / 경고: 주황 / 위험: 빨강), 범례 (정상/경고/위험 건수)
  - 센서 상세 페이지에서 현재 센서 ID에 해당하는 아이콘만 표시 (타 센서 아이콘 필터링) — v1.5.0
- **패널 드래그 리사이즈**: 좌측(센서 정보)↔중앙(평면도) 핸들 드래그 (160~360px)
- **시간별 트렌드** (v1.5.0 조회 버튼 추가)
  - **조회 버튼**: 날짜/조회단위/계산식/보정값/센서 선택 후 클릭 시에만 데이터 fetch
  - queryCondition state로 조회 조건 통합 관리 (즉시 fetch 방지)
  - 조회 기간: 오늘/7일/30일 프리셋 + 날짜 직접 입력
  - 조회 단위: 시간별 / 일별
  - **일별 모드**: 선택 시각(오전/오후 N시) 기준 데이터만 표시 — v1.5.0 시간 필터링 수정
  - 계산식 (Linear/Polynomial, 80053 전용), depth 선택 버튼 (아이콘 label 우선)
  - 측정값 카드 (기간 내 최신값/초기측정값/최솟값/최댓값)
  - 정상 구간 게이지 (현재값 위치 마커 표시)
  - 트렌드 차트
  - 보정값 입력 (80053 전용, depth별 독립 저장)
- **하단**: 측정 데이터 로그 테이블
  - 헤더에 아이콘 label 표시 (예: WL-01)
  - 시간별 모드: 1시간 단위 슬롯, 미수신 구간 `—` + `● 미수신`
  - 일별 모드: 선택 날짜/시각 기준 슬롯, 미수신 표시
  - 페이지네이션 (15건/페이지)

### 현장 상세 페이지 (`/sites/[id]`) — v1.5.0 신규
- 사이드바 현장 이름 클릭 → `/sites/N` (현장 카드 목록 우회, 바로 상세 뷰 이동)
- 사이드바 하단 "추가 및 편집" → `/sites` (기존 현장 관리 페이지)
- 센서 상세 페이지와 동일한 2단+트렌드+로그 레이아웃
- **좌측 패널**: 선택된 센서 기준 센서 정보
  - 관리번호/센서명/현장/설치위치/설치일/측정단위/측정주기/계산식/1차관리기준/마지막수신/계산식상수값
  - 소속 센서 목록 표시 (클릭으로 센서 전환)
  - MultiMonitor: 관리번호/센서명/현장/설치위치 4항목만 표시
- **평면도**: 아이콘 추가/수정/삭제/드래그, 평면도 업로드
  - MultiMonitor: 편집 불가 (읽기 전용)
  - 현장 단위 전체 센서 아이콘 표시 (센서 상세 페이지와 달리 전체 표시)
- **센서 추가/삭제**: 헤더 버튼, 센서 삭제 시 관련 평면도 아이콘도 함께 제거
- **시간별 트렌드**: 조회 버튼 클릭 시에만 fetch, queryCondition, 일별 시간 필터링
- **Excel/PDF 다운로드**: 헤더 버튼, 선택된 아이콘 라벨을 계측 No로 표시

### 차트 (SensorTrendChart) — v1.5.0 개선
- **ResizeObserver**: 컨테이너 크기 동적 감지, 차트 자동 크기 조정
- **가로 스크롤 방식**: 데이터 1건당 12px 고정 너비, 스크롤로 전체 구간 확인
- **데이터 끊김 표시**: null 구간에서 선 연결 없음 (connectNulls=false)
- **1차 기준선**: level1Upper/Lower prop으로 직접 수신 → depth 전환 시 즉시 반영
- 데이터 포인트 다이아몬드 마커 (데이터 200건 이하)
- Y축 소수 2자리 반올림
- 범례 (센서명/1차 하한기준/1차 상한기준)

### 80053 수위계 전용 기능
- **WL-02 평균값 처리**: depth 1/3번 데이터 평균으로 대체 (WL-02 DB 삭제) — v1.5.0
  - toHourKey() 함수로 KST 기준 시간 키 생성 (UTC→KST 변환 오류 방지)
- **depth_label 선택**: depth별 아이콘 label 버튼 (없으면 N번 수위계 fallback)
- **계산식 선택**: Linear(메인) / Polynomial 토글
- **depth별 보정값 입력**: depth별 독립적으로 보정값 입력 및 DB 저장 (correction_params)
- **depth별 1차 관리기준**: depth별 개별 상하한 설정 (depth_criteria DB 저장)
  - 센서 상세 페이지 인라인 편집 + 센서 편집 모달 양방향 동기화
  - 그래프 기준선, 측정값 상태, 엑셀, PDF 모두 depth별 기준값 반영
- psi→m 변환계수: 0.70307

### 엑셀/PDF — 계측 No 아이콘 라벨 표시 (v1.5.0)
- 평면도 아이콘 클릭 후 다운로드 시 계측 No에 아이콘 라벨(WL-01 등) 표시
- `iconLabel || sensor.manageNo` 우선순위로 적용
- 센서 상세 페이지, 현장 상세 페이지 모두 동일하게 적용

### 엑셀 출력 (Water Level Meter Report)
- **소수점 4자리 표시**: 측정값/전측정치대비/초기치대비 모두 toFixed(4) 적용
- 헤더 색상: 밝은 회색 (FFD9D9D9) 배경 + 검정 글씨
- Water Level Meter Report: 흰 배경 + 검정 굵은 글씨 (15pt)
- 라벨 글자 크기: 9pt / 내용값 글자 크기: 8pt
- 헤더 정보 (현장명/계측기 No.(아이콘 라벨 우선)/설치현황/초기측정일/관리자+권한/설치위치)
- 초기측정일: 전체 기간 최초 수신값 기준 (globalInitReading)
- 브라우저 차트 이미지 삽입 (SVG → PNG 변환, 6~15행)
- 범례: 센서명(파란) / 1차 하한기준(빨간) / 1차 상한기준(주황) 분리 표시
- **시간별 모드**: measurements(시간별 전체) 출력, 미수신 행 '미수신' 표시
- **일별 모드**: dailyReadings(선택 시각 데이터) 출력, 미수신 행 '미수신' 표시
- 초기치 행 강조 (노란 배경 + 빨간 텍스트)
- A4 인쇄 설정 자동 적용

### PDF 출력 (Water Level Meter Report)
- 헤더 정보 (관리자 이름+권한 포함, 계측기 No. 아이콘 라벨 우선)
- 초기측정일: 전체 기간 최초 수신값 기준 (globalInitReading)
- 차트: Y축 격자선/눈금, 데이터 포인트(◆), X축 레이블 (최대 6개)
- **depth별 1차 상한/하한 기준선**: DB 저장값 기준 (자동계산 제거)
- **데이터 끊김 구간 표시**: null 구간 선 연결 없음
- 범례: 가운데 정렬 (센서명/1차 하한기준/1차 상한기준)
- **시간별 모드**: measurementsWithGaps(null 포함) 출력
- **일별 모드**: dailyReadings 출력, 미수신 행 '미수신' 표시
- NanumGothic 한글 폰트 적용

- **현장 관리 토글**: 현장 이름 클릭 → `/sites/N` (현장 상세), 하단 "추가 및 편집" → `/sites`

### 현장 관리 (`/sites`)
- 현장 카드: 현장명/위치/설명/센서 현황/담당자
- 현장 편집 모달: 현장명/위치/설명/담당자/센서 배정/평면도 업로드
- **현장 추가 모달**: 평면도 업로드 비활성화 (현장 추가 후 편집에서 업로드 가능 안내)
- **평면도 현장 단위 통일**: 센서 상세 페이지와 현장 편집 모달이 동일한 현장 평면도 공유
- **현장 편집 저장 시 평면도 유지**: 담당자/이름 등 수정 후 저장해도 평면도 삭제되지 않음
- PDF 업로드 시 첫 페이지 PNG 자동 변환 저장 (pdf-to-png-converter)
- 별도 이미지 서빙 API로 모든 브라우저/환경에서 표시
- 페이지 헤더: "현장 추가 및 편집" (v1.5.0 명칭 변경)

### QR 기능
- **QR 모달**: 센서명 표시 (관리번호 대신)
- **QR 상세 페이지 열기**: 센서명으로 버튼 텍스트 표시
- **QR 상세 페이지 (`/qr/[id]`)**: 로그인 없이 센서 현재 상태 조회
  - 센서명 표시 (관리번호 제거)
  - 80053: depth별 아이콘 label 표시 + Linear(메인)/Polynomial 최신값
  - 일반 센서: 현재 측정값 표시
  - 보정값 적용된 최신값 표시
